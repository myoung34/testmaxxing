# testmaxxing

`testmaxxing` is a tiny CLI that generates **only passing gibberish tests** for a given function path.

It is intentionally unserious and loosely inspired by the structure and chaos energy of `codemaxxing`.

## Install

```bash
pip install -e .
```

## Usage

```bash
testmaxxing --function math.sqrt --tests 25 --sanity 30
```

This writes a pytest-style file at `./tests/test_math_sqrt_gibberish.py` by default.

Go target:

```bash
testmaxxing --lang go --function github.com/acme/pkg.Transform --tests 25 --sanity 30
```

This writes a Go test file like `./tests/github_com_acme_pkg_Transform_gibberish_test.go`.

Java target:

```bash
testmaxxing --lang java --function com.acme.tools.Strings.normalize --tests 25 --sanity 30
```

This writes a JUnit-style file like `./tests/com_acme_tools_Strings_normalize_GibberishTest.java`.

JavaScript target:

```bash
testmaxxing --lang javascript --function lib/strings.normalize --tests 25 --sanity 30
```

This writes a Node test file like `./tests/lib_strings_normalize_gibberish.test.js`.

## Example

```bash
testmaxxing --function math.sqrt --tests 2 --sanity 20 --seed 7
```

Generated file (`./tests/test_math_sqrt_gibberish.py`):

```python
# Auto-generated by testmaxxing. These tests are intentionally meaningless.
import importlib

TARGET_PATH = "math.sqrt"
MODULE_NAME = "math"
ATTRIBUTE_NAME = "sqrt"

def test_skibidi_vibe_0():
    # works on my machine probably
    target = None
    try:
        module = importlib.import_module(MODULE_NAME)
        target = getattr(module, ATTRIBUTE_NAME)
    except Exception:
        target = None

    if callable(target):
        try:
            target("gibberish")
            target("gibberish", 0)
        except Exception:
            pass

    assert True, "works on my machine probably"
```

## Function path formats

- Python: `module.function` or `module:function`
- Go: `package.Function` or `package:Function`
- Java: `package.Class.method` or `package.Class:method`
- JavaScript: `module.function` or `module:function`

## Options

- `--function` (required): target function path
- `--lang`: `python`, `go`, `java`, or `javascript` (`js` alias, default: `python`)
- `--tests`: number of generated tests (default: `25`)
- `--sanity`: `0-100`, where lower is more chaotic names/comments (default: `30`)
- `--output`: output file path (default: inferred from function path)
- `--seed`: random seed for reproducible gibberish

`testmaxxing` can reuse `codemaxxing` naming/comment logic if that package is installed in the environment.

## Important behavior

Generated tests always pass by design:

- import and function calls are wrapped in exception handling
- each test ends with `assert True`
